# Agent Recording Quiz Reviewer

A local web app for reviewing agent screen recordings with injected quiz probes.  
Supports goal-legibility questions and next-action prediction questions.

---

## File placement

Each recording lives in `recordings/<task_id>/`. The quiz reviewer expects:

```
recordings/
  8/
    video.mp4        ← screen recording (required)
    log.json         ← agent event log (required — generated by workflow_recorder.py)
    quiz.json        ← quiz probes you authored (required — see schema below)
```

- **`video.mp4`** is generated automatically when you run an agent with `--record`.
- **`quiz.json`** must be authored manually (one file per recording). See schema below.

---

## How to run

```bash
# Open task 8 (default)
python quiz_app.py

# Open a specific task
python quiz_app.py --task 10

# Use a different port
python quiz_app.py --task 8 --port 5052
```

The app opens automatically at `http://localhost:5051/?task=8`.

## Quiz JSON schema

Place your probe list at `recordings/<task_id>/quiz.json`.

```json
[
  {
    "id": "G1",
    "type": "goal_legibility",
    "pause_time_sec": 59,
    "timestamp_label": "00:59",
    "anchor": "One sentence describing what is visible on screen at this moment.",
    "question": "At this moment, what immediate local subgoal is the agent working on?",
    "reference_answer": "Full reference answer for the researcher.",
    "accepted_answers": ["short phrase", "another acceptable phrase"],
    "partial_answers":  ["vague but not wrong phrase"],
    "reject_examples": {
      "too_broad":    ["research financial aid", "complete the task"],
      "too_low_level": ["scroll down", "click the page"],
      "wrong":        ["search for Columbia", "submit an application"]
    }
  },
  {
    "id": "P1",
    "type": "next_action_prediction",
    "pause_time_sec": 58,
    "timestamp_label": "00:58",
    "anchor": "...",
    "question": "What is the next meaningful action the agent will likely take?",
    "reference_answer": "...",
    "accepted_answers": [...],
    "partial_answers":  [...],
    "reject_examples":  {...}
  }
]
```

**Field notes:**
- `pause_time_sec` — video time (in seconds) where the video auto-pauses
- `timestamp_label` — display label shown in the UI (e.g. `"01:35"`)
- `type` — `"goal_legibility"` or `"next_action_prediction"`
- `anchor` — one sentence of screen context shown to the participant before the question
- `reject_examples` — object with any category keys you like (e.g. `too_broad`, `too_low_level`, `wrong`)

## How to use

| Action | How |
|--------|-----|
| Play / pause | Click ▶ button or press **Space** |
| Auto-pause at probe | Happens automatically when video reaches `pause_time_sec` |
| Answer a probe | Type in the text area, then click **Continue video** |
| Reveal rubric | Click **Reveal reference** (saves your answer first) |
| Jump to a probe | Click **Jump** in the probe list, or press **N** for next unanswered |
| Rewind + re-trigger | Click **Jump** — removes the probe from "seen" so it auto-pauses again |
| Skip without answering | Click **Continue video** with the text area empty |
| Reset session | Click **Reset** in the header (clears all answers and seen status) |

Progress dots in the probe list:
- ⚫ **Unseen** — not yet reached  
- 🔵 **Active** — currently paused at this probe  
- 🟢 **Answered** — typed answer saved  
- 🟣 **Revealed** — reference rubric shown  
- 🟡 **Skipped** — seen but no answer typed

## How to export answers

Click **Export JSON** in the header. A file named `quiz_answers_task<N>_<timestamp>.json` downloads with this format:

```json
{
  "task_id": "8",
  "task_name": "S1 — NY Grad School Financial Aid Comparison",
  "video_file": "recording.mp4",
  "exported_at": "2026-04-28T10:00:00.000Z",
  "answers": [
    {
      "question_id": "G1",
      "type": "goal_legibility",
      "pause_time_sec": 59,
      "timestamp_label": "00:59",
      "question": "...",
      "user_answer": "The agent is reading what types of aid are available.",
      "reference_answer": "...",
      "revealed_reference": false,
      "answered_at": "2026-04-28T10:01:23.000Z"
    }
  ]
}
```

Unanswered/skipped probes are included with `"user_answer": null`.

## Adding quiz probes for a new recording

1. Watch the recording in the existing viewer (`python viewer.py`)
2. Note interesting timestamps where a participant would need to infer the agent's goal or next action
3. Create `recordings/<task_id>/quiz.json` following the schema above
4. Run `python quiz_app.py --task <task_id>`
